What is FlowMCP?

FlowMCP is an open-source framework for turning existing web APIs into a standardized Model Context Protocol (MCP) interface. It allows AI assistants and applications to access web API data and services in a structured, consistent manner. By acting as a universal adapter between AI and external APIs, FlowMCP makes it easier to bring real-world data (especially from Web3/blockchain services) into AI-driven conversations and tools.

In practical terms, FlowMCP provides a declarative schema system: each API integration is defined by a schema that describes the API’s endpoints, parameters, and how to handle responses. These schemas enable an MCP-compatible server to present API calls as “tools” that an AI can invoke. The result is seamless and testable AI integration for hundreds of routes across various services.

Key Features

  • Declarative Schemas for APIs: Each API route is defined in a schema file (module) with its endpoint, parameters, and description. This ensures all tools are self-documented and AI-readable.
  • Pre/Post Processing Hooks: Schemas can include optional modifiers that run before or after the API call. These allow data normalization or custom logic (e.g. formatting responses) without altering the core API.
  • Built-in Testing: Every schema can contain test cases for its routes. This automated testing guarantees that tools function as expected and helps catch issues early when APIs change.
  • Human-Readable Results: API responses are returned as clean, text-based outputs. Even error messages are descriptive, which helps the AI (and developers) understand the results easily.

Usage & Schema Library

Using FlowMCP: To use FlowMCP, you run an MCP server (local or remote) and load the desired API schemas into it. The FlowMCP library provides utilities to activate schemas on a server instance with minimal code. Once the server is running, any AI client that supports MCP can connect to it and start calling the available tools.

Example: A developer can run a local MCP server and activate the CoinGecko schema. The AI assistant could then answer questions like “What’s the price of Bitcoin?” by invoking the getCoinPrice tool from the CoinGecko schema, fetching live data and returning it as a friendly text answer. The developer didn’t need to manually code how to call CoinGecko – FlowMCP’s schema handled it.

Schema Library: FlowMCP comes with a rich library of schemas (over 200 API endpoints) covering many popular Web3 and crypto data providers. These include:
Blockchain data & analytics (e.g. Etherscan, Moralis, Dune Analytics), Cryptocurrency info (e.g. CoinGecko, CoinMarketCap, CoinCap), DeFi and NFT data (e.g. Chainlink feeds, Dexscreener, The Graph), and more. By leveraging this library, you can immediately empower your AI with knowledge of these services. And if something’s missing, you can add your own schema!

Note: FlowMCP is in active development. In upcoming releases, we plan to provide a one-command setup for a local server and a hosted remote server option for convenience (see below).

Contributing & Future Plans

Contributing: We welcome contributions from developers and enthusiasts. If you have an idea for a new schema or improvement, please visit our GitHub repositories. You can fork the FlowMCP-schemas repo to add new API schemas – just drop in a new .mjs file for your provider and submit a pull request. For changes to the core framework or for reporting issues, head to the FlowMCP core repo. We appreciate tests and documentation with contributions to maintain quality.

Future Roadmap: The project is gearing up to introduce a user-friendly Local Server package, allowing anyone (even non-developers) to run FlowMCP tools on their own machine with minimal setup. Additionally, a secure Remote Server deployment is planned, so that a cloud-based FlowMCP service could serve multiple clients over the internet. These enhancements will broaden the ways FlowMCP can be used – from personal offline setups to shared online services. Stay tuned for updates!
